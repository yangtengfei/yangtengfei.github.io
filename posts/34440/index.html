简单工厂

发表于 2018-12-12 | 阅读次数：

1. 概述

1.1 定义：由一个工厂对象决定创建出哪一种产品类型的实例
1.2 类型：创建型，但不属于GOF23种设计模式。
1.3 适用场景：
1.3.1 工厂类负责创建的对象比较少
1.3.2 客户端（应用层）只传入工厂类的参数，对于如何创建对象（逻辑）不关心
1.4 优点：只需要传入一个正确的参数，就可以获取所需要的对象，无须知道创建细节。
1.5 缺点：工厂类的职责相对过重，增加新的产品需要修改工厂类的判断逻辑，违背了开闭原则。

2. Code

在该部分的代码中，包含工厂类VideoFactory，抽象类产品Video，产品JavaVideo\PythonVideo和测试类Test，测试类代表更上层的应用层。
2.1 抽象类Video如下：

public class JavaVideo extends Video{
@Override
public void produce() {
System.out.println(“录制Java 课程视频”);
}

}

public class PythonVideo extends Video{
@Override
public void produce() {
System.out.println(“录制Python课程视频”);
}

}

public class Test {
public static void main(String[] args) {
Video video = new JavaVideo();
video.produce();
}
}

public class VideoFactory {
public Video getVideo(String type) { // 也可以是静态方法，方便直接调用；但是不方便继承、修改
if (“java”.equalsIgnoreCase(type)) {
return new JavaVideo();
}else if (“Python”.equalsIgnoreCase(type)) {
return new PythonVideo();
}
return null;
}
}

public class Test {
// 简单工厂
public static void main(String[] args) {
VideoFactory videoFactory = new VideoFactory();
Video video = videoFactory.getVideo(“java”);
if (video == null) {
return;
}
video.produce();
}
}

public class VideoFactory {
// 使用反射，增加扩展性
public Video getVideo(Class class1) {
Video video = null;
try {
video = (Video)Class.forName(class1.getName()).newInstance();
} catch (Exception e) {
System.out.println(“exception”);
}
return video;
}
}

VideoFactory videoFactory = new VideoFactory();
Video video = videoFactory.getVideo(JavaVideo.class);
if (video == null) {
return;
}
video.produce();
}
}
